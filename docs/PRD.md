# Design & roadmap

`@ancplua/typespec-maf` generates Microsoft Agent Framework (MAF) declarative YAML
from TypeSpec, and proves the output by loading it through Microsoft's own agent
and workflow loaders.

## Why

MAF has hand-authored YAML dialects with no typed source:

| | Dialect | Loader |
|---|---|---|
| D1 | prompt-agent YAML (`kind: Prompt`) | `ChatClientPromptAgentFactory.CreateFromYamlAsync` |
| D2 | workflow YAML (`kind: Workflow`) | `DeclarativeWorkflowBuilder.Build` |
| D3 | Foundry hosted manifests | Foundry hosting |

Hand-authored, they share the usual failure modes: untyped strings, hand-managed
ids and cross-references, PowerFx with no compile-time checking, and no schema gate
before the runtime failure. TypeSpec gives them one typed source: this library
emits the YAML, and a test loads each artifact through Microsoft's own loader, so
what it produces is YAML MAF actually accepts. Microsoft built
[`@microsoft/typespec-m365-copilot`](https://www.npmjs.com/package/@microsoft/typespec-m365-copilot)
for M365 Copilot; this applies the same approach to the Agent Framework.

## Principles

1. **Round-trip or it doesn't ship.** An emitter change is mergeable only with the
   Microsoft-loader round-trip test green against the pinned MAF version.
2. **Generated action shapes.** D2 action models are generated by reflecting over
   the pinned `Microsoft.Agents.ObjectModel`, never hand-written. Preview churn is a
   pin bump + regenerate + diff review.
3. **Errors at compile, not load.** Unknown action references and undeclared PowerFx
   variables are compile errors here, where raw MAF fails only at load.
4. **Pinned versions.** MAF packages and the TypeSpec toolchain are pinned; no
   floating `latest`.

## Status

- [x] **D1 — prompt-agent YAML** — `@agent` / `@instructions` / `@useModel` / `@tool`,
      round-trip via `CreateFromYamlAsync`.
- [x] **Action-shape generator** — `tools/maf-model-gen` reflects over the pinned
      ObjectModel into `generated/maf-actions.gen.tsp`.
- [x] **D2 — workflow YAML** — `@workflow`, reference-id validation, PowerFx reference
      validation, round-trip via `DeclarativeWorkflowBuilder.Build`.
- [ ] **D3 — Foundry hosted manifests** (`@hosted`, `maf-hosted` emitter).
- [ ] **Telemetry declaration + emit.**
- [ ] **Release to npm.**

## Architecture

```text
ObjectModel (pinned MAF pkg) ──► tools/maf-model-gen ──► generated/maf-actions.gen.tsp
                                                              │
 lib/  @agent @instructions @useModel @tool @workflow          │ (D2 action shapes)
       powerfx reference validation                           ▼
 lib/src/index.ts emitter — *.agent.yaml (D1)   *.workflow.yaml (D2)   maf-hosted (D3, planned)
       │                              │
       ▼                              ▼
 ChatClientPromptAgentFactory   DeclarativeWorkflowBuilder   ◄── round-trip test harnesses
```

See [`AGENTS.md`](../AGENTS.md) for the repo layout and the build/test workflow.
